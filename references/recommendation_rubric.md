# recommendation_rubric.md
# 论文推荐评分规则

当有多篇候选论文时，用这个评分框架决定推荐哪几篇、推荐顺序是什么。

---

## 五个评分维度

### 1. Relevance（相关性）
**问题**：这篇论文和用户的具体问题/场景有多匹配？

| 分数 | 描述 |
|------|------|
| 3 | 直接处理用户的核心问题，场景高度吻合 |
| 2 | 相关但有一定距离，需要用户自己做迁移 |
| 1 | 大方向相关，但具体问题对不上 |
| 0 | 关键词相关但实质不匹配 |

**注意**：相关性不是"这篇论文有名"，而是"这篇论文和用户的问题有多近"。

---

### 2. Readability（可读性）
**问题**：对这个用户来说，这篇论文好不好读？

| 分数 | 描述 |
|------|------|
| 3 | 写作清晰，即使非专业读者也能理解核心内容 |
| 2 | 需要一些背景知识，但关键部分可以读懂 |
| 1 | 技术性很强，需要大量专业背景 |
| 0 | 对目标用户来说几乎不可读 |

**注意**：可读性和用户角色有关。同一篇论文，对研究生可能是3分，对刚接触领域的老师可能是1分。

---

### 3. Pedagogical Richness（教学设计丰富度）
**问题**：这篇论文对理解学习机制或改进教学设计有多大帮助？

| 分数 | 描述 |
|------|------|
| 3 | 有清晰的学习机制解释 + 可操作的教学或设计建议 |
| 2 | 有学习机制或教学建议，但不完整 |
| 1 | 有结论但缺乏机制解释或设计指导 |
| 0 | 纯技术/算法论文，无教学视角 |

---

### 4. Productizability（产品转化潜力）
**问题**：这篇论文的设计或发现，能不能转化为产品功能或交互设计？

*仅在用户是 builder / 产品设计者时计入评分*

| 分数 | 描述 |
|------|------|
| 3 | 有具体的系统设计、交互流程或反馈机制，可直接参考 |
| 2 | 有设计启发，但需要较多转化工作 |
| 1 | 结论有参考价值，但没有设计层面的抓手 |
| 0 | 和产品设计基本无关 |

---

### 5. Evidence Strength（证据强度）
**问题**：这篇论文的结论有多可信？

| 分数 | 描述 |
|------|------|
| 3 | 随机对照试验 / 系统综述 / Meta 分析 |
| 2 | 准实验研究 / 多案例对比 / 大样本调查 |
| 1 | 单案例研究 / 小样本实验 / 相关性研究 |
| 0 | 纯观点 / 无数据支持 |

---

## 评分使用规则

### 标准推荐（Guided 模式）

计算加权分：
```
总分 = Relevance × 3 + Readability × 1.5 + Pedagogical Richness × 1.5 + Evidence Strength × 1
```

优先推荐总分最高的3篇。

### Builder 模式

计算加权分（Productizability 权重加倍）：
```
总分 = Relevance × 3 + Readability × 1 + Pedagogical Richness × 1 + Productizability × 2 + Evidence Strength × 1
```

### 研究验证模式

Evidence Strength 权重加倍：
```
总分 = Relevance × 2 + Readability × 1 + Evidence Strength × 3
```
同时要求：推荐3篇中，至少1篇对用户假设有反驳或挑战。

---

## 推荐顺序规则

1. **第一篇**：Relevance 最高的论文（最贴近用户问题）
2. **第二篇**：补充第一篇没有覆盖的维度（可能是更强的证据，或更清晰的设计）
3. **第三篇**：拓展视角（相邻方向，或对第一篇结论的挑战）

---

## 不推荐的红线

无论评分多高，以下情况不推荐：

- **无法确认存在**：如果不确定论文是否真实存在，不推荐，改为给关键词和搜索方向
- **只讲模型性能**：如果论文只报告 accuracy / F1 等指标，对用户无实质帮助
- **场景严重错位**：如果论文研究的是完全不同的人群或情境
- **超过10年且领域快速发展**：AI 教育领域，超过5-7年的论文要标注"可能有更新研究"

---

## 快速判断工具（给 Claude 用）

```
推荐前自问：
□ 我确定这篇论文存在吗？
□ 这篇和用户问题的连接是什么？（一句话说清楚）
□ 它比其他候选论文更适合这个用户的理由是什么？
□ 用户读这篇时应该重点看哪里，可以跳过哪里？
□ 这篇的局限是什么？

如果任何一个问题答不上来，不推荐这篇。
```
